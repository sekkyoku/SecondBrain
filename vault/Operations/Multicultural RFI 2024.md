## Data Collection & Methodology

| Question | Answer |
|----------|--------|
| Data type | **Proprietary, Deterministic** |
| Data sources | Canela 1P (digital apps/websites), monetization data (Springserve, ad servers), 3P (LiveRamp) |
| Identity graph | Device ID (CTV ID, MAID) |
| Total unique devices | **76MM** |
| % MAIDs | **38%** (Canela Universe) |
| Logged-in users | <5% (no authentication required; rewards program expected to increase) |
| Annual churn rate | **~60%** |
| Content volume | **35,000 hours** |
| Content crawling | Ad-serving based — each ad request makes video content eligible |
| Data opt-in | App opt-in + ad server opt-in |
| Data refresh | Some daily, some monthly |
| Cost structure | CPM, Flat Fees |

### Data Validation

- Source verification from trusted/vetted sources
- Anomaly detection (automated: missing values, duplicates, spikes)
- Cross-system reconciliation (periodic)
- Audience cross-validation against LiveRamp
- Manual reviews and audits
